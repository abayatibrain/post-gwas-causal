# ADR-0002: Reimplement coloc.abf in Python rather than calling R `coloc`

Date: 2026-06-17
Status: Accepted

## Context

Colocalization via `coloc.abf` (Giambartolomei 2014) is a small, well-specified
algorithm: per-SNP ABFs for two traits, combined into five hypothesis
posteriors under priors `p1, p2, p12`. The canonical implementation is the R
`coloc` package. We must decide whether the toolkit *calls* R or *reimplements*
the method in Python.

## Options considered

### Option A — Shell out to R `coloc` for every analysis
- **Pros:** uses the reference implementation directly; no risk of subtle math
  divergence.
- **Cons:** makes R a hard runtime dependency for a core feature; brittle in CI
  and on machines without R; slow process-spawn per call; opaque to Python
  users and harder to unit-test against known answers.

### Option B — Reimplement `coloc.abf` in Python, keep R as an *optional* cross-check
- **Pros:** zero hard R dependency; fast, vectorised, fully type-hinted, and
  serializable (pydantic result). The algorithm is small enough to implement
  exactly. R `coloc` is retained in `scripts/R/coloc.R` purely to *validate*
  the Python output (`requires_r` test).
- **Cons:** we own the correctness; must guard numerical edge cases (the H3
  off-diagonal log-sum-exp when one SNP dominates).

## Decision

**Option B.** `coloc/abf.py` is a from-scratch Python `coloc.abf`. R `coloc` is
optional and used only as a reference oracle through `rbridge.py`.

## Consequences

- No R needed to run colocalization, the CLI, or CI.
- Correctness is pinned by known-answer tests (PP.H4 > 0.8 under a shared
  causal; PP.H3 dominant under distinct causals; posteriors sum to 1) and by an
  optional Python-vs-R agreement test that skips when `Rscript` is absent.
- The H3 term is computed via the stable identity
  `offdiag = full − diag` in log space, with the diagonal clipped away from the
  full sum to avoid `log1p(-1)` when a single SNP carries all the mass.
